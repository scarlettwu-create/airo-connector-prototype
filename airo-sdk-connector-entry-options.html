Create connector entry point options

Four ways to place the "build with AIRO" vs "build manually" choice on the custom connectors page, from the neutral fork you have today to a full AI-first prompt. Each is a live mockup so you can see how the two actions actually sit.

The real decision is hierarchy: do you treat the two paths as equal, or make AIRO the front door and keep manual as the escape hatch? Equal-split is the safe default but it undersells AIRO and taxes every user with a choice. If AIRO is the right-to-win, lean to Option 2 or 3.

Pros
  • Neutral; doesn't bias users who already know what they want
  • Simplest to build; what you have now
  • Manual path is fully visible, no one feels funnelled
Cons
  • Adds a decision tax on every connector creation
  • Treats a power-user escape hatch as equal to the path you want to grow
  • Undersells AIRO; the "or" implies a 50/50 toss-up
Use when you genuinely have no opinion on which path should win, or manual users are the clear majority and would resent a nudge.

Placement: one centered input as the hero. The manual path is a single text link below it, so only the user who wants it has to opt out.
Pros
  • Collapses the fork into one obvious action
  • Most honest expression of AIRO as the right-to-win
  • Matches v0, Figma Make, Replit, where the input is the product
  • Lets the user start typing intent immediately, no extra click
Cons
  • Boldest change; biggest shift from today's page
  • Manual path is visibly de-emphasized, needs buy-in
  • Empty input can feel daunting without good placeholder/examples
Use when leadership is committed to AIRO as the front door. Pair it with starter examples so the blank box never stalls a new user.
4

Single entry + mode toggle

No decision tax

One "Create connector" surface that opens in AIRO mode by default, with a toggle to switch to manual. Removes the upfront fork entirely.

Placement: a segmented toggle at the top of one create surface. AIRO is the pre-selected tab; manual is one click away in the same place.
Pros
  • Zero upfront decision; you land in the default and start
  • Both modes live in one consistent surface
  • Easy to change the default later without redesign
Cons
  • Manual path is one layer less discoverable than a visible card
  • Needs a clear create entry point first (button/page) to host the toggle
  • Two modes in one surface adds state to maintain
Use when you want to eliminate the choice screen and have a natural "Create connector" button to open this from.
